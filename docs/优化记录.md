# 性能优化记录

> 日期：2026-06-18
> 目标：提升立创商城（LCSC）批量查询速度，保证查询成功率

---

## 改动清单

### 1. HTTP/1.1 持久连接（替代每次新建短连接）

**文件**：`bom_tool/adapters/lcsc_adapter.py`

- 新增 `_html_client` 持久的 `httpx.AsyncClient` 实例
- 新增 `_get_html_client()` — 懒创建，HTTP/1.1 keep-alive，连接池 3 路
- 新增 `_do_http_get()` — 包装 GET 请求，TransportError 时自动重建 + 重试
- 修改 `_do_html_request()` — 从 `async with httpx.AsyncClient(...)` 改为复用持久客户端
- 更新 `close()` — 关闭 `_html_client`

### 2. Akamai 反爬规避（核心可靠性保障）

**文件**：`bom_tool/adapters/lcsc_adapter.py`

- **主动轮换**：`_HTML_CLIENT_MAX_REQUESTS = 3` — 每 3 次查询重建 TCP 连接，在 Akamai 开始挑战前更换
- **被动检测**：响应 < 50KB 时识别为挑战页面，自动重建连接 + 重试（安全网）
- 计数重置：`_recreate_html_client()` 将 `_html_request_count` 归零

### 3. 速率限制可配置化

**文件**：
- `bom_tool/adapters/registry.py` — `create_adapters()` 新增 `lcsc_interval` 参数
- `bom_tool/cli.py` — 新增 `--lcsc-interval` 参数（默认 1.2）
- `bom_tool/ui/main_window.py` — 新增 `lcsc_interval` 贯穿 `PipelineWorker` → `SingleQueryWorker`

### 4. 速度优先模式

**文件**：`bom_tool/ui/main_window.py`

- GUI"速度优先"模式：`max_concurrent=2`、`lcsc_interval=0.8`
- GUI"稳定优先"模式：`max_concurrent=1`、`lcsc_interval=1.2`
- `precheck_cache()` 也跟随查询模式设置 LCSC 间隔

### 5. 缩短重试退避

**文件**：`bom_tool/adapters/lcsc_adapter.py`

- `attempt * 4 + random(0.5, 1.5)` → `attempt * 2 + random(0.3, 0.8)`

## 根因分析

Akamai 边缘防护会对**同一 TCP 连接上的第 4 个请求起**返回 10KB 挑战页面（无实际产品数据）。

**三个关键修复**：
1. **轮换阈值 8 → 3**：在 Akamai 开始挑战前主动重建连接
2. **挑战页面检测**：响应 < 50KB 时自动重建连接并重试（安全网）
3. **连接断开恢复**：TransportError 时自动重建连接并重试（兜底）
