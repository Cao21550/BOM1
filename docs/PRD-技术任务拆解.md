# BOM表自动完善工具 PRD + 技术任务拆解

## 1. 文档信息

- 产品名称：BOM表自动完善工具
- 当前版本：MVP 需求版
- 目标平台：Windows 桌面端
- 技术路线：Python + PySide6 + asyncio + httpx + openpyxl + SQLite
- 主要用户：电子工程师、采购人员、硬件项目管理人员

## 2. 产品目标

本工具用于读取用户本地 BOM 文件，识别用户指定的器件型号或商城 SKU 列，批量从立创商城、华秋商城、一博在线等供应商平台查询库存、价格、描述、数据手册等信息，并以另存为新文件的方式回填到原 BOM 表中，同时尽可能保留原文件格式。

MVP 的核心目标不是覆盖所有商城和所有 Excel 复杂格式，而是完成一条稳定的端到端流程：导入文件、配置映射、查询数据、展示进度、生成新文件。

## 3. 用户与使用场景

### 3.1 电子工程师

- 希望快速确认 BOM 中器件是否有货。
- 希望找到对应数据手册链接。
- 希望减少手工复制商城信息的时间。

### 3.2 采购人员

- 希望批量获取库存、单价、阶梯价和购买链接。
- 希望对比不同供应商报价。
- 希望输出表格仍能直接用于内部采购流程。

### 3.3 项目管理人员

- 希望快速估算物料可采购性。
- 希望识别缺货、高价、无匹配结果的物料。

## 4. MVP 范围

### 4.1 必须支持

- 支持 `.xlsx` 文件读取、预览、回填并另存。
- 支持 `.csv` 文件读取和导出。
- 支持用户选择搜索列。
- 支持按器件型号 MPN 查询。
- 支持至少一个真实供应商 Adapter 端到端查询。
- 支持供应商插件化扩展。
- 支持型号清洗。
- 支持异步批量处理和进度回调。
- 支持 SQLite 缓存。
- 支持基础 PySide6 桌面界面。

### 4.2 MVP 暂缓

- `.xls` 旧格式完整保样式回填。
- 海外平台 Mouser、DigiKey。
- SaaS 云端版本。
- 团队协作。
- 自动登录、验证码处理。
- 复杂 Excel 多工作表批量同时处理。

## 5. 功能需求与完成标准

### 5.1 文件导入与预览

需求：用户可以导入 BOM 文件，系统读取工作表、表头和前 20 行数据。

完成标准：

- 能成功读取 `.xlsx` 的第一个工作表。
- 能成功读取 `.csv`，自动处理 UTF-8、UTF-8-SIG 常见编码。
- UI 中能展示表头和前 20 行预览。
- 文件不存在、文件损坏、格式不支持时给出明确错误。
- 不修改原文件。

### 5.2 动态列映射

需求：用户从表头中选择搜索关键字列，并选择搜索方式。

完成标准：

- 下拉框展示所有有效表头。
- 支持搜索类型：器件型号、商城 SKU、自动识别。
- 用户未选择搜索列时不能开始批量任务。
- 重复表头显示列序号辅助区分。
- 映射结果可转换为业务配置对象。

### 5.3 型号清洗

需求：提升搜索命中率，去除常见包装后缀和多余字符。

完成标准：

- 输入为空时返回空结果且不抛异常。
- 保留原始型号。
- 输出清洗后型号。
- 支持去除 `-TR`、`-CT`、`-ND`、`-T`、`-T&R` 等常见后缀。
- 单元测试覆盖大小写和空格场景。

### 5.4 供应商查询 Adapter

需求：供应商抓取逻辑插件化，核心任务管理器不依赖具体商城实现。

完成标准：

- 定义 `BaseSupplierAdapter` 抽象基类。
- 每个 Adapter 支持 `search_by_mpn` 和 `search_by_sku`。
- Adapter 输出统一的 `PartResult` 结构。
- 单个供应商失败不影响其他供应商。
- 查询失败时返回标准状态和错误原因。

### 5.5 批量查询任务

需求：系统批量处理 BOM 行并持续更新进度。

完成标准：

- 支持并发数配置，默认 5。
- 支持每个关键字查询多个供应商。
- 支持进度回调，包含总数、完成数、成功数、失败数、缓存命中数。
- 单行失败不终止整个任务。
- 支持任务取消接口。

### 5.6 SQLite 缓存

需求：减少重复查询，提高响应速度。

完成标准：

- 缓存键包含供应商、搜索类型、关键字。
- 缓存值保存标准化结果 JSON。
- 支持过期时间，默认 24 小时。
- 支持强制刷新绕过缓存。
- 缓存命中能反映到进度统计中。

### 5.7 Excel 原格式回填

需求：查询结果写入新文件，尽量保留原 BOM 格式。

完成标准：

- 使用 `openpyxl.load_workbook` 打开原文件。
- 不覆盖原文件，只另存为新路径。
- 新增列写入所选字段。
- 原有单元格样式、列宽、行高、合并单元格不主动破坏。
- 新增列表头样式复制相邻表头样式。
- 保存失败时提示可能原因，例如文件被占用或无权限。

### 5.8 CSV 输出

需求：CSV 不涉及样式保留，但需要追加查询字段。

完成标准：

- 能读取原 CSV。
- 输出新 CSV。
- 原始列顺序不变。
- 查询字段追加在末尾。
- 保持 UTF-8-SIG 编码，便于 Excel 打开。

### 5.9 单次查询

需求：用户输入一个型号或 SKU，立即展示多个供应商结果。

完成标准：

- 输入为空时禁用查询。
- 可选择供应商。
- 结果展示库存、价格、描述、商品链接、数据手册链接。
- 查询中有加载状态。
- 失败时展示可读错误。

### 5.10 设置

需求：用户可以配置影响查询和输出的常用参数。

完成标准：

- 可配置并发数。
- 可配置请求超时。
- 可配置缓存有效期。
- 可配置是否启用缓存。
- 可配置代理地址。
- 设置能保存到本地配置文件。

## 6. 非功能需求

### 6.1 性能

- 500 行 BOM 在网络正常时应可稳定处理。
- UI 不因批量查询阻塞。
- 缓存命中查询应在毫秒级返回。

### 6.2 稳定性

- 单个供应商异常不能导致整个程序崩溃。
- 单个物料异常不能中断批量任务。
- 所有外部请求必须设置超时。

### 6.3 可扩展性

- 新增供应商只需要新增 Adapter 并注册。
- 核心任务管理器只依赖抽象接口。
- 标准字段变化集中在数据模型层处理。

### 6.4 安全与隐私

- 文件默认只在本地处理。
- 不上传完整 BOM 到自有服务。
- 日志不记录敏感路径之外的无关隐私信息。

## 7. 标准数据字段

`PartResult` 建议字段：

| 字段 | 含义 | 必填 |
| :--- | :--- | :--- |
| supplier | 供应商名称 | 是 |
| query | 原始查询词 | 是 |
| search_type | mpn / sku / auto | 是 |
| status | success / not_found / failed | 是 |
| mpn | 匹配型号 | 否 |
| sku | 商城编号 | 否 |
| brand | 品牌 | 否 |
| package | 封装 | 否 |
| description | 描述 | 否 |
| stock | 库存 | 否 |
| moq | 最小起订量 | 否 |
| price_unit | 单价 | 否 |
| price_breaks | 阶梯价 | 否 |
| lead_time | 交期 | 否 |
| product_url | 商品链接 | 否 |
| datasheet_url | 数据手册链接 | 否 |
| confidence | 匹配置信度 | 否 |
| error_message | 错误信息 | 否 |
| fetched_at | 查询时间 | 是 |

## 8. 技术任务拆解

### 阶段一：项目骨架

- 创建 Python 包结构。
- 创建 `pyproject.toml`。
- 创建核心目录：`core`、`adapters`、`db`、`ui`、`utils`、`tests`。
- 定义数据模型。
- 定义 Adapter 抽象接口。

验收标准：

- 能执行 `python -m bom_tool.main`。
- 所有包可正常 import。
- 基础单元测试能启动。

### 阶段二：文件处理

- 实现 Excel 读取。
- 实现 CSV 读取。
- 实现表头提取。
- 实现数据预览。
- 实现 Excel 回填。
- 实现 CSV 导出。

验收标准：

- 样例 `.xlsx` 可读取并生成新文件。
- 原文件未被修改。
- 新增列写入结果字段。

### 阶段三：查询核心

- 实现型号清洗。
- 实现任务管理器。
- 实现缓存模块。
- 实现 Mock Adapter。
- 接入至少一个真实供应商 Adapter。

验收标准：

- 命令行单次查询能返回标准 JSON。
- 批量查询能输出进度。
- 重复查询命中缓存。

### 阶段四：桌面 UI

- 实现主窗口。
- 实现批量处理页。
- 实现单次查询页。
- 实现设置页。
- 接入后台任务线程。

验收标准：

- 用户可通过 UI 完成导入、配置、查询、导出。
- 查询期间窗口可操作。
- 错误信息展示清晰。

### 阶段五：测试与发布

- 增加单元测试。
- 增加 100 行和 500 行样例压测。
- 配置 PyInstaller 打包。
- 编写用户使用说明。

验收标准：

- Windows 可执行文件可在干净环境启动。
- 样例文件端到端流程通过。
- 发布包包含必要说明文档。
