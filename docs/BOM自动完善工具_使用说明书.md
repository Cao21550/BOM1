## 1. 软件概述

BOM 表自动完善工具是一款 Windows 桌面工具，用于批量查询电子元器件供应商的库存、价格、规格等信息，并将结果自动回填到 BOM（物料清单）表中。

**支持供应商：**

| 供应商 | 说明 |
|--------|------|
| 立创商城 (LCSC) | 网页搜索，实时查询库存、价格、品牌、封装、数据手册 |
| 华秋商城 (HQChip) | 网页搜索 + API 兜底，实时查询库存、价格、品牌、封装、数据手册 |

**核心功能：**

- 读取 `.xlsx` / `.csv` BOM 文件
- 自动识别 BOM 表头（型号列、封装列等）
- 批量查询多个供应商
- SQLite 缓存减少重复查询
- 保留原 Excel 格式另存为新文件
- 单次快速查询单个型号

## 2. 系统要求

- **操作系统**：Windows 10 / Windows 11（64位）
- **内存**：最低 512MB，建议 2GB 以上
- **硬盘空间**：200MB 以上
- **网络**：需要互联网连接（访问供应商网站）

## 4. 批量处理

### 4.1 基本流程

```
选择 BOM 文件 → 读取表头 → 选择搜索列 → 选择供应商 → 开始回填 → 输出新文件
```

### 4.2 操作步骤

1. **选择文件**：点击「选择文件」按钮，选取 `.xlsx` 或 `.csv` 格式的 BOM 文件
2. **读取表头**：点击「读取表头」自动解析，预览表格显示前 20 行
3. **选择搜索列**：在预览表格中点击列头，或从下拉框选择；系统会自动优先选择"型号""MPN"等列
4. **选择工作表**：如 Excel 有多个工作表，可从下拉框切换
5. **选择供应商**：勾选需要查询的商城（立创商城、华秋商城）
6. **设置查询参数**：

| 参数 | 说明 |
|------|------|
| 查询方式 | 器件型号（MPN）或 商城编号（SKU） |
| 查询模式 | 稳定优先（慢但稳）或 速度优先（快但有风险） |
| 并发数 | 同时查询的线程数（建议 1-2） |
| 缓存 | 启用后重复查询直接读取本地缓存 |
| 缓存有效期 | 缓存保留的小时数（默认 168h = 7天） |
| 自动重试 | 失败项自动补跑 |

7. **开始回填**：点击「开始回填」，进度条和日志会实时显示查询进展
8. **查看结果**：程序自动生成带 "_supplier_completed" 后缀的新文件

### 4.3 查询模式详解

| 模式 | LCSC 间隔 | 首轮并发 | 适用场景 |
|------|:---------:|:--------:|---------|
| **稳定优先** | 1.2s | 1 | 首次查询、网络不稳定、大批量 |
| **速度优先** | 0.8s | 2 | 缓存命中率高、网络状况良好 |

### 4.4 其他操作

- **预检查缓存**：提前显示哪些型号已缓存、哪些需要联网
- **重试失败项**：批量完成后单独重试失败的型号
- **暂停/取消**：查询过程中可暂停或取消任务
- **清空缓存**：清除所有历史缓存数据

## 5. 单次查询

### 5.1 操作步骤

1. 切换到「单次查询」标签页
2. 输入器件型号或商城编号
3. 选择查询方式（器件型号 / 商城编号）
4. 勾选要查询的供应商
5. 点击「查询」按钮
6. 结果显示在下方表格中
7. 点击任意单元格可复制内容

### 5.2 缓存说明

单次查询也使用 SQLite 缓存。同一型号的重复查询会直接从缓存返回（毫秒级响应），无需重复请求网络。

## 6. 命令行模式

```powershell
# 基本用法：立创商城查询
python -m bom_tool.cli 输入BOM.xlsx 输出BOM_回填.xlsx --search-column 型号 --supplier lcsc

# 多供应商查询
python -m bom_tool.cli 输入BOM.xlsx 输出BOM_回填.xlsx --search-column 2 --supplier lcsc --supplier hqchip

# 编译版用法
BOM自动完善工具_20260619.exe 输入BOM.xlsx 输出BOM_回填.xlsx --search-column 型号 --supplier lcsc
```

**CLI 参数说明：**

| 参数 | 说明 |
|------|------|
| `input` | 输入文件路径（必需） |
| `output` | 输出文件路径（必需） |
| `--search-column` | 搜索列：表头名或 1 开始的列号（必需） |
| `--search-type` | 搜索类型：`mpn`（默认）或 `sku` |
| `--supplier` | 供应商：`lcsc`、`hqchip`；可重复 |
| `--max-concurrent` | 并发数（默认 1） |
| `--field` | 输出字段，可重复 |
| `--no-cache` | 禁用缓存 |
| `--cache-ttl-hours` | 缓存有效期（默认 168） |
| `--lcsc-interval` | LCSC 请求间隔秒数（默认 1.0） |

## 7. 输出字段说明

批量处理后，原文件会追加以下列（每供应商一组）：

| 字段 | 含义 | 示例 |
|------|------|------|
| `供应商_状态` | 查询结果状态 | success / not_found / failed |
| `供应商_型号` | 匹配到的型号 | STM32F103C8T6 |
| `供应商_编号` | 商城商品编号 | C8323 |
| `供应商_品牌` | 器件品牌 | ST(意法半导体) |
| `供应商_封装` | 封装类型 | LQFP-48 |
| `供应商_描述` | 商品描述 | 32位ARM Cortex-M3微控制器 |
| `供应商_库存` | 当前库存数量 | 19200 |
| `供应商_最小起订量` | 最小订购数量 | 1 |
| `供应商_单价` | 单价 | 12.50 |
| `供应商_阶梯价` | 批量价 | 1:12.50, 10:10.00 |
| `供应商_商品链接` | 商品页 URL | https://... |
| `供应商_数据手册` | 数据手册 URL | https://... |
| `供应商_错误信息` | 失败原因 | HTTP 429 Too Many Requests |

## 8. 缓存机制

### 8.1 工作原理

- 缓存存储在 `.cache/bom_cache.sqlite3` 文件中
- 缓存键 = 供应商 + 搜索类型 + 关键字
- 默认有效期 7 天（168 小时）
- 缓存命中时零网络开销，毫秒级返回

### 8.2 管理缓存

- GUI：勾选/取消「启用 SQLite 查询缓存」
- GUI：点击「清空缓存」按钮删除所有缓存
- CLI：使用 `--no-cache` 跳过缓存
- CLI：使用 `--cache-ttl-hours N` 调整有效期

## 9. 常见问题

### Q1: 查询结果全是失败/未匹配

可能原因：
- 网络连接问题，检查能否访问供应商网站
- AKAMAI 反爬挑战，稍后再试
- 型号格式过于特殊，尝试简化型号（去除括号内容）

### Q2: 程序启动后窗口空白

- 确保系统安装了合适的显卡驱动
- 尝试以管理员权限运行

### Q3: 杀毒软件报毒

- PyInstaller 打包的程序属于常见误报
- 添加信任/排除项即可

### Q4: Excel 文件格式损坏

- 确保输入文件为 `.xlsx`（非 `.xls`）
- 用 Excel 打开后另存为 `.xlsx` 格式再试
- 避免使用 WPS 特殊格式

### Q5: 网速很慢，查询时间长

- 立创商城受 Akamai CDN 保护，部分型号需要 7-10 秒
- 切换到「速度优先」模式
- 减少并发数到 1，避免被限流
- 下次查询同一型号会走缓存，速度快很多

### Q6: 如何排查问题

- 查看界面中的日志输出区域
- 日志包含每个查询的状态、耗时、错误信息
- 检查 `.cache/gui_stderr.log` 和 `.cache/gui_stdout.log`
